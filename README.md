## 🧠 About The Project

This is a **CRUD demo project built to learn and apply CQRS with Clean Architecture** in ASP.NET Core.

The main focus of this project is not business complexity, but **understanding how to structure backend applications in a scalable and maintainable way**.

---

## 🎯 Key Goals

- Understand and apply **CQRS pattern (Command & Query Separation)**
- Learn how to structure projects using **Clean Architecture**
- Practice separating concerns between layers
- Build a clean and maintainable CRUD API structure
- Improve backend design thinking

---

## 🏗️ Architecture

The project follows Clean Architecture principles:

- **Domain Layer** → Core business entities  
- **Application Layer** → CQRS handlers, DTOs, interfaces  
- **Infrastructure Layer** → Database access using EF Core  
- **API Layer** → Controllers / Endpoints  

---

## ⚙️ Features

- Create, Read, Update, Delete operations
- CQRS separation for commands and queries
- DTO mapping for clean API responses
- Layered architecture for maintainability
- Basic validation and structured request handling

---

## 🛠️ Tech Stack

- ASP.NET Core Web API  
- C#  
- Entity Framework Core  
- SQL Server  
- CQRS Pattern  
- Clean Architecture  

---

## 📚 What I Learned

- How CQRS separates read/write responsibilities
- How Clean Architecture improves project structure
- How to organize large-scale backend applications
- Importance of separation of concerns
- Writing maintainable and testable code

## 🚀 Purpose

This project was built as a **learning sandbox to strengthen backend fundamentals and system design thinking** before moving to real-world complex systems like ERP and multi-tenant applications.
